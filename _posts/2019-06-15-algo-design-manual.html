Algorithm Analysis

Growth Rates


  • Constant: f(n) = 1
  • Logarithmic: f(n) = log(n)
  • Linear: f(n) = n
  • Superlinear: f(n) = n * log(n)
  • Quadratic: f(n) = n^2
  • Cubic: f(n) = n^3
  • Exponential: f(n) = c^n
  • Factorial: f(n) = n!

    • Data Structures

    Arrays:
  • Constant-time access, given an index
  • Space efficiency
  • Memory locality

    • Linked Lists:
  • No overflows (unless memory is physically full)
  • Simpler insertions & deletions
  • Requires space for pointers
  • No efficient random access

    • Stacks
  • LIFO: Push, Pop operations

    • Queues
  • FIFO: Queue, Dequeue operations

    • Dictionaries
  • Permits access by content
  • Search, Insert, Delete operations
  • Max, Min, Predecessor, Successor operations (some types of Dicts)
  • Array operation costs - unsorted vs sorted:
  • Dictionary operation costs - sorted/unsorted, single/double links:

    • Binary Search Trees

  • Search, Min element, Max element, Traverse (all elements), Insert, Delete operations
  • Ops can take O(height) time if tree is perfectly balanced. But insertions can cause trees to be unbalanced very easily.

    • Priority Queues
  • Allows new elements to be placed at arbitrary intervals = more flexibility.

    • Hashing
  • Hash: a math function that maps keys to integers. Use hash outputs as array indexes.
  • Collisions: two distinct keys can return identical hash values.
  • Chaining: most common way to resolve collisions. Natural solution, but uses lots of pointer memory.

  • Open Addressing: if desired position is already occupied, use another position (often n+1 or n-1)

  • Operation times (m-element hash table, doubly-linked chain lists)


    • Sorting & Searching

    Factors:
  • Increasing or decreasing order?
  • Sorting the key or entire record?
  • Equal keys. Now what?
  • Non-numerical data. Now what?

    • Heapsort

    A heap-labeled tree = a binary tree such that the key of each node "dominates" the keys of its children. You can store a binary tree in an array w/o pointers - space efficient, but missing nodes take up unused space.

    Mergesort

    Partition elements into two groups, sort each smaller problem recursively, then interleave two sorted lists to order the merged elements.

    Quicksort


    Distribution Sort

    Ex: sorting phone book by 1st letter of last name ==> 26 buckets. Repeat with 2nd letter of last name (smaller piles), etc. Done when each bucket contains a single name.

    Effective when distribution is roughly uniform, terrible when not.
